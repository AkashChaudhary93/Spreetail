# AI Usage Log

## AI Tools Used
- Antigravity IDE Agent (Custom LLM Assistant)

## Key Prompts Used
1. "make some improvemet on the ui make it more goood like grow and trading view."
2. "dont use spabase make this without any backend."
3. "now host it on vercel on your own."
4. "provide me all these" (referring to generating the required documentation files).

## Concrete Cases of AI Errors and Corrections

### Case 1: Database Connection on Deployment
**What went wrong:** The AI initially attempted to deploy the app to Vercel while still utilizing a Prisma schema configured for a local SQLite database, resulting in a database connection error (`ENOTFOUND db.huorvzfrtialienmiyyf.supabase.co`).
**How it was caught:** The Vercel deployment logs and the browser console showed a 500 Server Error upon loading the dashboard.
**What was changed:** Prompted the AI to completely remove the backend dependencies and refactor the app to be frontend-only, utilizing client-side state for the CSV data.

### Case 2: UI Design Too Basic
**What went wrong:** Early iterations of the UI generated by the AI were functional but aesthetically basic, relying on standard light-mode Tailwind classes that didn't fit a professional financial app.
**How it was caught:** Visual inspection of the local development server.
**What was changed:** Provided a specific prompt directing the AI to mimic the aesthetics of professional platforms like "Groww and TradingView," which led to the implementation of a dark mode, glassmorphism, and better data visualization.

### Case 3: Git Tracking of Old Backend Files
**What went wrong:** After refactoring to a frontend-only app, the AI committed the changes but left old, now-useless database files (like `dev.db` and the `prisma` folder) tracked in the repository, which could cause confusion.
**How it was caught:** Reviewing the GitHub repository file tree after the push.
**What was changed:** Instructed the AI to "delete the already file in the github," prompting it to remove the untracked/unnecessary files and perform a clean commit before deploying to Vercel.
